### 页面正文
在开始写上一篇文章 Agentic SOC 时，我就在想有没有一个Agentic SOC 平台，能够对接各种安全设备、原生 agent，能够完成：把告警、上下文、工具调用、证据整理和初步研判组织成一条可执行流程。
Flocks [1]的设计理念正好适合这件事。Flocks 是一个原生的 Agentic SOC 的平台：支持多 agent、workflow 执行引擎、各种类型的设备接入等。
Flocks 最关键的地方，是它不是只提供一个 agent，而是把多 agent 能力和 workflow 执行引擎放在同一个平台里。agent 可以调用 workflow 执行，workflow 也可以调用 agent，可以灵活编排使用。
workflow 负责流程稳定，agent 负责探索、处理不确定性。
多 agent 解决的是“不确定性”。比如一条告警进来后，需要分析它像不像真实攻击、需要补哪些上下文、当前证据支持哪些假设、有没有反证、下一步该查什么。这类问题很难完全写成固定规则，更适合让 agent 结合工具和上下文去做调查。
workflow 执行引擎解决的是“确定性编排”。比如告警接入、字段归一化、规则过滤、去重、结果落盘、发 Kafka、发通知。这些步骤不应该每次都让 agent 自由发挥，而应该被固化成可执行、可追踪、可复用的流程。
综上，对于“AI Agent 应该先查证据、补上下文、排除噪声、缩小假设空间”[2]这个观点，Flocks 给出的不是一个单点 Agent，而是一套可以把告警接入、流程编排、工具调用和 Agent 研判串起来的平台化答案。
既然 Flocks 已经提供了丰富的设备数据接入、workflow、agent 执行引擎，那我们就开始按下面的流程实现

真正开始做之前，我们需要先准备三类能力。
第一类是告警入口。Flocks 的 workflow 需要能接到真实告警流，比如 Kafka、Syslog 接收来自 EDR、NDR、防火墙、WAF、SIEM 的告警输出。这里重点是把原始告警数据接进来。
第二类是设备和日志查询能力。要让 Agent 查证据，必须具备连接设备的能力。比如 Network Agent 要能查 http、代理日志；Endpoint Agent 要能查 EDR 进程树、命令执行、文件落地。
第三类是 IOC 和上下文查询能力。IP、domain、hash、URL、CVE、资产归属。它们应该通过 Flocks 的工具、MCP 或 API 接入威胁情报、漏洞库、CMDB、工单系统。

第一步由 Triage Agent 做调查规划。它先看告警类型、资产等级、已有上下文和历史相似事件，初步判断这条告警更像扫描、攻击尝试、攻击成功、账号异常、数据外传，还是已知噪声。生成候选假设和调查路径：哪些问题交给 Network Agent，哪些问题交给 Endpoint Agent，哪些问题需要 Identity Agent 或 Enrichment Agent 补充。
Network Agent 负责网络侧证据。它查外联、DNS、代理日志、NDR 会话、同网段相似行为、横向移动迹象。比如外联域名是否新注册、是否命中情报、是否有周期性 beacon、是否多个主机访问同一可疑地址。
Endpoint Agent 负责主机侧证据。它查进程树、命令执行、父子进程关系、文件落地、启动项、脚本执行、EDR 行为日志。比如可疑外联是否由浏览器、业务进程、PowerShell、未知二进制或计划任务触发。
Identity Agent 负责身份侧证据。它查账号、权限、登录、token、MFA 异常、VPN、堡垒机和云 IAM。比如告警发生前后是否有异地登录、MFA 失败、权限提升、token 异常使用。
Enrichment Agent 负责补业务上下文。它查资产、业务、负责人、系统等级、是否公网、是否有历史事件、近期是否变更。很多告警本身看不出严重性，必须知道这台机器属于什么业务、谁负责、是否承载关键数据。

这里的重点是“可复核的候选假设和建议”，不是“最终结论”。看到这份 case 后，应该能快速知道：为什么它可能有风险，为什么它也可能只是噪声，以及哪些动作必须人工确认。
workflow 根据 Response Agent 的结构化输出，按“置信度 + 影响范围”做三类动作分流。
建议动作 = 自动执行：只适用于高置信度、低影响动作，比如标记重复告警、关闭低风险重复噪声。建议动作 = 待人工确认：适用于中置信度或中影响动作，Agent 给出证据和建议，分析师确认后再执行。建议动作 = 待人工研判：适用于低置信度或高影响动作，比如禁用账号、吊销 token、隔离主机、阻断网络、升级重大事件，Agent 只辅助调查和整理证据。比如历史上反复确认过的失败扫描、无成功响应、无主机异常、目标资产不受影响、命中明确白名单的重复告警，可以进入自动关闭候选。
