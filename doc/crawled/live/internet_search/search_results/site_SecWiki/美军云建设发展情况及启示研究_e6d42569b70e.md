# 美军云建设发展情况及启示研究

### 页面正文
美军围绕JADC2全域联合作战建设目标，立足多军种差异化作战需求，从网络架构、云环境部署、基础设施布局3个方面统筹推进军事云体系建设，形成架构分层弹性、部署分级分类、防护全域纵深的建设模式。
2.1 分层弹性架构与全域互联能力结合的云网络体系
美军云网络采用分层分布式设计，贯通战略指挥至战术执行全链路，形成功能互补、全域互联的弹性支撑体系，共同支撑全域作战云的网络接入与数据传输需求。
2.1.1 分层网络架构：适配全域作战场景
美军云网络架构通过核心骨干网络、区域网络与战术边缘网络3个层级，构建适配全域作战场景的弹性通信支撑体系。
（1）核心骨干网络。连接本土战略级数据中心与全球主要军事基地，以光纤与天基通信为主，承担跨战区数据调度与全球云资源协同，是JADC2/CJADC2 的基础支撑。
（2）区域网络。部署于各战区及重点区域，采用光纤与区域卫星通信融合模式，运用软件定义网络（software defined network, SDN）技术实现带宽动态分配，利用网络功能虚拟化（network functions virtualization, NFV）技术实现硬件功能软件化部署，可根据作战任务需求快速完成网络拓扑重构。
（3）战术边缘网络。面向前沿单兵、车辆、无人装备等终端，融合无线自组网、5G、卫星通信等手段，依托战术5G基站、自适应跳频与“星链”（Starlink）低轨星座，支撑高清战场态势回传，保障复杂电磁与机动环境下通信的连续性。
2.1.2 关键技术应用：融合创新与实战化验证
美军以作战需求为牵引，推进SDN/NFV、5G、卫星通信等技术融合应用，实现云网络能力的跨越式提升。
（1）SDN与NFV协同应用。SDN通过分离控制平面与数据平面，依托集中式控制器实现全域网络统一管理与资源动态调度；NFV将路由器、防火墙等传统硬件功能封装为虚拟网络功能（virtual network function, VNF）。二者协同实现区域骨干网与战术边缘网快速部署、弹性扩展、多密级通信隔离等关键需求。
（2）5G与军事云深度融合。重点攻克低延迟网络切片、抗干扰传输等核心技术，提供安全可靠的全域网络接入服务，契合JADC2/CJADC2跨域数据互通的核心目标。JWCC云平台依托战术云持续扩大覆盖范围，逐步实现全球关键作战区域的覆盖。
（3）卫星通信升级。通过高轨升级、低轨组网、终端小型化提升卫星通信效能，支撑军事云全域覆盖。在高轨领域，推进演进战略卫星通信系统（evolved strategic satcom, ESS）研发，计划最终部署6颗静止轨道卫星（geostationary orbit satellite，GEO）和高椭圆轨道卫星（highly elliptical orbit satellite，HEO），提升战略通信容量与抗干扰能力；在低轨领域，美国太空发展局（Space Development Agency，SDA）加快“运输层”（Transport Layer）卫星星座部署，构建低延迟、高弹性战术通信网络；2025年9月，通过 SpaceX猎鹰9号火箭成功发射21颗作战部署1批次（Tranche 1）卫星，标志其正式从技术验证阶段转入实战化应用部署阶段；在终端方面，同步推进战术卫星通信终端（tactical satellite communication terminal，TSCT）的小型化、轻量化与模块化改造，提升前沿作战单元云接入能力。
2.1.3 安全防护措施：全域纵深防御与主动验证结合
美军构建了多层次的云网络安全防护体系，通过身份认证、数据加密、智能监测与自动化响应的协同应用，保障云网络安全与稳定。
（1）部署零信任架构。摒弃传统网络边界信任模型，将零信任理念融入云网络安全体系，对所有接入用户、设备实施持续身份认证与权限验证，相关架构已在“层云”（Stratus）私有云部署应用。
（2）强化全链路加密与认证。传输层采用高级加密标准（advanced encryption standard，AES）256与椭圆曲线密码（elliptic curve cryptography, ECC）加密，存储层以AES-256为主，试点应用同态加密。应用层试点采用属性基（attribute based encryption, ABE）加密，实现细粒度权限管控，配合多因素认证（multifactor authentication, MFA）提升访问安全。
（3）构建智能监测与响应体系。实时监测云网络流量；运用大数据与AI构建异常行为检测模型；依托安全编排自动化与响应（security orchestration, automation and response, SOAR）平台实现威胁处置自动化。
2.2 多元架构协同与分层适配密级需求的云环境部署
美军采用私有云、公有云、混合云协同架构，并构建海外双云体系，形成适配不同密级与场景的分层部署体系。
（1）私有云。作为高密级业务的核心载体，私有云由美军自主建设运维，部署于涉密设施，通过多级安全认证（multi-level security, MLS）承载机密级以上业务。“层云”已在夏威夷、日本及欧洲等海外基地部署，可支持高带宽武器平台数据的实时整合，以及高密级数据处理与资源共享。
（2）公有云。公有云成本效益高，资源弹性扩展能力强，适用于安全性要求相对较低的非涉密业务，如科研项目数据处理、模拟训练、后勤管理等。美军引入亚马逊网络服务公司（Amazon Web Services, AWS）、微软Azure等商业云服务，通过“联邦风险与授权管理计划”（Federal Risk and Authorization Management Program, FedRAMP）认证与数据物理隔离保障安全。
（3）混合云。混合云融合了私有云的安全性与公有云的弹性优势，可按业务灵活分配资源，是支撑美军全域作战的核心云架构。如在军事演习中，部分模拟训练等业务依托公有云开展，以利用其算力；核心作战计划、机密情报等则存储于私有云，以保障安全。在联合作战边缘云（joint operations edge cloud, JOE）中，前沿传感器数据就近在边缘节点预处理，核心指挥数据在私有云存储运算，大规模推演任务则调用公有云算力，实现商用云技术与战术边缘环境的高效适配。
（4）海外双云协同体系。美军面向海外作战构建“层云”与JOE双云架构，融合战略级全局支撑与战术边缘实时响应能力，适配通信受限环境下的海外作战需求。“层云”部署于夏威夷、日本及欧洲等海外基地，提供多军种弹性计算共享与高带宽武器平台数据整合；JOE机动部署于航母战斗群与前沿阵地，专注战术数据就近处理与低时延响应，通过弹性调度提升战场数据处理效率。印太战区凭借地理与战略价值，成为该体系的关键试验场。
2.3 分级部署与边缘前置协同的云设施建设布局
美军云设施围绕数据中心分级、边缘计算前置、软硬件军标适配建设，保障全域算力持续供给。
2.3.1 数据中心：分级部署与冗余设计结合
美军在全球范围内构建了庞大的数据中心网络，通过地理分散部署、功能分级协同及差异化功能定位与冗余设计，保障云设施的抗毁性与服务连续性。
（1）战略级。主要部署于本土，采用多区域冗余部署，在弗吉尼亚州、科罗拉多州等地建设核心节点，配备电磁脉冲防护（electromagnetic pulse protection, EMP）、生物特征门禁、备用发电机集群等安全保障设施，承载核心军事数据与战略指挥应用。
（2）战区级。部署于各战区核心区域，为区域内作战行动提供本地化算力。采用模块化预制建设模式，建设周期较传统数据中心大幅缩短。如印太战区数据中心位于日本横田、韩国乌山、澳大利亚等地，支撑印太威慑战略指挥决策需求。
（3）战术级。靠近作战前沿，为一线部队提供实时数据处理与存储服务。通常采用模块化、可移动设计，可快速部署与灵活调整。例如，美陆军战术级数据中心可集成在移动方舱内，支持数百个战术终端接入；海军则将战术数据中心集成于舰艇平台，支撑航母战斗群远洋作战云应用需求。
2.3.2 边缘计算设施：算力前置与异构环境适配
随着作战需求向战术边缘延伸，美军将边缘计算设施前置，通过车载、机载、分布式节点部署模式，下沉算力至战术前沿，满足无人机控制、战场态势实时感知等低时延作战需求。
（1）车载/机载边缘计算单元。集成于主战装备如车辆、飞机等的嵌入式算力单元，采用低功耗高性能芯片架构，具备抗振、抗高温、电磁兼容特性，适配装备机动过程中的复杂环境。如美陆军“斯特赖克”装甲车搭载的战术边缘计算节点（tactical edge computing node, TECN）可对战场环境数据进行快速融合分析；美空军F-35战斗机集成机载边缘云模块，实现目标识别算法实时迭代，大幅缩短了空战决策周期。
（2）分布式边缘节点。部署于前沿阵地的小型化算力站点，采用太阳能与蓄电池供电，支持无人值守运行，通过自组网、卫星通信等方式互联，形成分布式边缘计算网络，实现数据就近预处理与云端回传。美军已在叙利亚、伊拉克等海外热点地区部署多个分布式边缘节点，成为反恐行动中的前端感知与计算枢纽。
2.3.3 软硬件配置：军标适配与开源定制结合
以满足军事场景需求为核心，通过高性能硬件选型与开源、定制化软件组合，保障云设施的计算能力、存储效率与军事适配性。
（1）硬件。服务器以高性能x86架构服务器为主体，特殊场景采用专用芯片定制化服务器；部署图形处理单元/张量处理单元（graphics processing unit/tensor processing unit, GPU/TPU），满足大模型训练、战场图像识别、情报数据挖掘等高强度算力需求；采用固态硬盘（solid state drive, SSD）实现高速存储；配备高速交换机与加密路由器，保障高性能计算、大规模数据存储与安全传输。
（2）软件。 采用开源与定制化结合模式，操作系统以定制化Linux为主，保留部分Windows Server，以兼容专用军事应用。云管理平台基于OpenStack、VMware vSphere等构建，针对性扩展多密级资源隔离与全域调度能力，实现跨域云资源统一管控；数据库采用关系型与非关系型混合架构，Oracle、SQL Server支撑指挥类结构化数据存储，MongoDB、Redis适配传感器、作战日志等非结构化数据，保障高效存储与查询；同时自主研发作战指挥辅助、情报分析引擎等专用软件，并与云平台深度集成。
